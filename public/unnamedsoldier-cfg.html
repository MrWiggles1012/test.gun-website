Unnamedsoldier.cfg

Find, edit, and protect your MOHAA config file (bonds, CVARs, and more).

Translations Available:
English | Arabic | German | Russian | Armenian | Spanish | Dutch



Unnamedsoldier.cfg is your main personal configuration file — it stores your custom settings and preferences.
You’ll usually find it in the configs folder:

  • Spearhead: C:\Games\MOHAA\mainta\configs\
  • Breakthrough: C:\Games\MOHAA\maintt\configs\

This file contains things like your binds (key binds), CVARs (game settings), and other options that control how the game looks, feels, and performs.
It’s basically the place where you “tune” the game to your liking.

Tip: Before you edit anything, it’s a good idea to make a quick backup copy of the file (for example: unnamedsoldier_backup.cfg). That way you can always restore your original settings if something goes wrong.

If you can’t find unnamedsoldier.cfg, launch the game once and close it again — the file is created automatically after the first run.


Protect unnamedsoldier.cfg from unwanted changes

Some servers (or server-side settings) may change your config file and “mess up” your personal settings.
To protect your setup, you can set unnamedsoldier.cfg to Read-only.

  1. Right-click unnamedsoldier.cfg.
  2. Select Properties from the context menu.
  3. In the General tab, find the Attributes section.
  4. Check the Read-only box.
  5. Click Apply, then OK.

Tip: If you want to edit or add something later, temporarily disable Read-only, make your changes, save the file, and then enable it again.


Remove forced aliases (Foresight and similar tools)

In some cases, Foresight (or other third-party tools) may automatically add certain aliases to your config file.
These “trap” aliases are sometimes used to block cheat commands by forcing the game to quit, or by making you send an embarrassing message in chat.

If you didn’t add these lines yourself, it’s safe to treat them as unwanted. They can also cause confusion later when a command suddenly doesn’t work as expected.

Examples:

alias wb_item_esp "quit"
alias r_loadmap "say I'm trying to crash the server! KICK ME! KICK ME!"

Tip: Before removing anything, make a quick backup copy of your config file (for example: unnamedsoldier_backup.cfg).
Then delete the alias lines, save the file, and restart the game.


Clean up other config files

You can also remove other config files in the same folder (for example player.cfg), because they can sometimes overwrite or conflict with unnamedsoldier.cfg.


Optional: Lock down foresight.cfg

If you see a foresight.cfg file in the same folder, you can stop it from altering your settings by clearing it and locking it:

  1. Open foresight.cfg with Notepad (or any text editor).
  2. Delete everything inside the file so it’s completely empty.
  3. Save the file.
  4. Set the file to Read-only as well.

This helps prevent servers running Foresight from changing your configuration.

unnamedsoldier.cfg is het configuratiebestand met al je persoonlijke instellingen. Je vindt het in mohaa\main\configs.

Het bevat je binds (toetsen), CVARs (instellingen) en andere opties die bepalen hoe je speelt.

Bescherm unnamedsoldier.cfg tegen ongewenste wijzigingen

Sommige servers kunnen dit bestand aanpassen of “verknallen”. Om dat te voorkomen kun je het bestand op Alleen-lezen (Read-only) zetten.

  1. Klik met rechts op unnamedsoldier.cfg
  2. Kies Eigenschappen
  3. Vink Alleen-lezen aan
  4. Klik op Toepassen

Wil je later iets wijzigen? Zet Alleen-lezen tijdelijk uit, pas je config aan en zet Alleen-lezen daarna weer aan.

Verwijder geforceerde aliases (Foresight en vergelijkbaar)

Het kan gebeuren dat Foresight (of een ander programma) aliases toevoegt. Dit wordt soms gebruikt om cheat-commando’s te blokkeren door de game te laten afsluiten of door je in chat te laten “bekennen”.

Voorbeelden:

alias wb_item_esp "quit"
alias r_loadmap "say I'm trying to crash the server! KICK ME! KICK ME!"

Deze aliases kunnen conflicteren met MOHAAC (en mogelijk andere anti-cheats), dus het is aan te raden om ze uit unnamedsoldier.cfg te verwijderen.

Andere configbestanden opschonen

Je kunt ook andere configbestanden in dezelfde map verwijderen (zoals newconfig.cfg of player.cfg), omdat ze instellingen kunnen overschrijven of conflicten kunnen veroorzaken met unnamedsoldier.cfg.

Optioneel: foresight.cfg “locken”

Zie je een foresight.cfg in de map? Dan kun je voorkomen dat servers met Foresight je instellingen blijven aanpassen:

  1. Open foresight.cfg in Notepad
  2. Verwijder alles in het bestand
  3. Sla het bestand op
  4. Zet het ook op Alleen-lezen
